# 벤치마크 난이도 설계 — 난이도 factor 개관

이 문서는 bm1~bm5 가 **공유하는 뼈대**와, 각 벤치마크가 조절하는 **난이도
factor** 를 정리한다. 개별 벤치마크의 상세 구성은 `bm1_easy.md` ~
`bm5_epistasis.md` 참조.

핵심 아이디어 하나: **다섯 벤치마크는 전체 난이도를 비슷하게 유지하되,
"지배적 난이도 요인"만 다르게** 만든다. 그래서 "어떤 optimizer 가 이기는가"가
벤치마크마다 달라지고, 그 차이가 곧 알고리즘의 강점 지도를 그린다.

---

## 0. 공통 파이프라인 — X 에서 raw_y 까지

모든 벤치마크가 똑같이 거치는 4단계다. 벤치마크마다 다른 것은 **②의 latent
함수뿐**이고, ①③④는 베이스 클래스(`BenchmarkBase`)가 공유한다.

```
X (30 signed 정수)
  ─①─▶ u = space.to_unit(X)          # 각 컬럼 [0,1] 등간격, 값 표현과 무관
  ─②─▶ latent = _latent(u)  → (6,)    # 벤치마크 고유 지형. 방향 무관 "값 자체"
  ─③─▶ 은닉 스케일/노이즈               # 스칼라 목적에만 가산 가우시안
  ─④─▶ raw_y = 관측 장치 렌더           # 타원 마스크 2장 + 스칼라 2개
```

### 6개 목적과 블록 구조 (공통)

- **블록**: `common`(col 0–9, 10개) · `set1`(10–14, 5개) · `set2`(15–29, 15개).
- **목적 의존성**: y11·y12·y13 → group1(common+set1) / y21·y22·y23 →
  group2(common+set2). `set1 ⫫ set2 | common` — 두 그룹은 common 을 통해서만 엮인다.
- **방향(sense)**: 최대화 y11,y12,y21,y22 / **최소화 y13,y23**.
- latent 벡터 인덱스: `[y11, y12, y13, y21, y22, y23]`.

### ④ 관측 장치 (구조화 raw_y) — 전 벤치 공통

latent 6값을 숫자로 그대로 내보내지 않고 **측정 장치의 원형**으로 렌더한다:

| latent | 관측 형태 | 측정치(optimizer 의 convert_y_raw) |
|---|---|---|
| latent[0], latent[1] | `mask1` (128×128 bool 타원) | y11 = max height, y12 = max width |
| latent[3], latent[4] | `mask2` (128×128 bool 타원) | y21 = max height, y22 = max width |
| latent[2] (y13) | 스칼라 float | 그대로 (은닉 스케일 적용) |
| latent[5] (y23) | 스칼라 float | 그대로 |

- **타원 렌더**: latent → 반축 픽셀(`_semi_px`, latent 클수록 큰 타원) →
  가우시안 필드 `G = exp(−½((r/σr)²+(c/σc)²))`, σ=반축/√(2 ln 2) 로 잡아
  `G ≥ 0.5` 등고선이 정확히 그 반축의 타원이 되게 함.
- **노이즈**: 타원 = 경계 밴드(|G−0.5|<0.08) 픽셀을 35% 확률로 flip(+격자
  양자화) — 가산 가우시안이 아니라 **경계 너덜거림**. 스칼라 = 주효과 std 의
  5% 가우시안. `noisy=False` 면 완전 결정적.
- 이 관측 모델은 난이도와 **직교**한다 — 지형(난이도)은 latent 가 정하고,
  관측 장치는 "값이 어떻게 측정되어 나오는가"만 바꾼다.

### ③ 은닉 스케일 — "값 범위 사전정보 없음" 강제

스칼라 목적은 `y = latent·scale + offset` 으로 은닉 스케일을 씌운다.
j그룹별 규격(j=1 수천, j=2 ~1 음수오프셋, j=3 ~0.0x)에 ±20% 지터.
목적 간 스케일이 5~6자릿수 차이 나므로 **온라인 robust 스케일러 없이는
비교 자체가 불가능**하다. (타원 목적은 픽셀 단위라 스케일이 자연히 통일됨.)

## 1. 난이도 factor 6종

각 벤치마크는 아래 성분을 latent 에 더하거나 빼서 지형을 조각한다.
성분마다 **어떤 탐색 전략을 저격하는지**가 다르다.

| # | factor | 수식 성분 | 무엇을 어렵게 하나 | 저격 대상 |
|---|---|---|---|---|
| F1 | **단봉 골격** | `Σ w·(1−(u−peak)²)` | (기본 지형 — 쉬움) | — |
| F2 | **블록 내 상호작용** | `Σ ±0.05·u_a·u_b` (같은 블록 쌍) | 변수쌍 결합 | 좌표 독립 가정 |
| F3 | **교차-블록 상호작용** | `Σ w·cos(π(u_common−u_set))` | common 이 바뀌면 set 최적이 이동 | 블록 분해("common 먼저 고정") |
| F4 | **기만(deception)** | 넓은 가짜 오르막 + 좁은 진짜 정상 | 국소 정보가 틀린 방향을 가리킴 | greedy / 1-hop 언덕오르기 |
| F5 | **ruggedness** | `Σ r·sin(15~25·u)` 고주파 저진폭 | 미세 개선 신호가 노이즈에 묻힘 | 유한차분·민감한 수락판정 |
| F6 | **목적 충돌(trade-off)** | `+c_gain·c(u)`, c=common 가중평균 | 전 목적 동시만족 해가 없음 | 스칼라화·단일 최적 |

각 factor 를 컬럼 2개에 대한 3D 서피스로 본 것: [`factors_3d.png`](factors_3d.png).
F1 돔(분리가능) / F2 안장(비분리 시작) / F3 대각선 능선(블록결합) / F4 가짜고원+모서리
급봉(기만) / F5 계란판 잔물결(rugged) / F6 기울어진 평면(전 목적에 같은 부호로 가산).

각 부품의 1D 모양은 [`primitives.png`](primitives.png), 5벤치의 실제 지형을
3D 서피스(y11 latent 를 컬럼 2개에 대해)로 본 것은
[`landscapes_3d.png`](landscapes_3d.png) 참조.

**F6(목적 충돌)이 특히 중요하다.** trade-off 축 c 는 common 블록의 가중평균인데,
c 가 커지면 **모든** latent 가 커진다. 최대화 목적(y·1,y·2)엔 이득이지만
최소화 목적(y·3)엔 손해 → 한 해가 6목적을 동시에 만족시킬 수 없다. 이게
chebyshev(최악목적 지배) 점수의 상한을 1보다 한참 아래로 누르는 장치다.
c_gain 을 키우면 상한이 내려가고, 없애면 "전부 좋은 해"가 생겨 벤치가 포화한다.

## 2. 벤치마크별 factor 배합

전체 난이도(단봉 골격 대비 난이도 성분의 분산 기여)는 비슷하게 맞추되,
**지배 요인만 다르다.** trade-off·노이즈·스케일 규격은 5벤치 공통.

| factor | bm1 easy | bm2 medium | bm3 hard | bm4 deceptive | bm5 epistasis |
|---|---|---|---|---|---|
| 단봉 골격 계수 | 1.0 | 1.0 | 0.7 | 0.7 | **0.5** |
| F2 블록 내 쌍 | — | 8쌍 | 8쌍 | 4쌍(약) | **12쌍(강)** |
| F3 교차-블록 | — | — | 10쌍(약) | — | **20쌍(강)** |
| F4 기만 | — | — | 약(3컬럼 g) | **진짜 trap 6컬럼** | — |
| F5 rugged | — | — | 있음 | — | — |
| F5' 저주파 다봉 | — | sin 진폭 0.15 | — | — | — |
| F6 trade-off gain | 0.25–0.35 | 0.3–0.45 | 0.35–0.5 | 0.35–0.5 | 0.35–0.5 |
| **지배 요인** | 없음(분리가능) | 완만한 다봉 | 복합 | 기만 | epistasis |

## 3. 실측 결과 (budget 780, 11 opt × 10 seed, pooled chebyshev)

각 벤치의 챔피언과 random 을 대조하면 factor 배합이 실제로 작동함이 보인다.

| 벤치 | 챔피언 | random | 갭 | 판별력 |
|---|---|---|---|---|
| bm1_easy | xgb_tr 0.755 | 0.440 (11위) | 0.315 | 정상 |
| bm2_medium | xgb_tr 0.833 | 0.485 (11위) | 0.348 | 정상 |
| bm3_hard | xgb_tr 0.777 | 0.433 (11위) | 0.344 | 정상 |
| bm4_deceptive | xgb_tr 0.775 | 0.452 (11위) | 0.323 | 정상 |
| bm5_epistasis | xgb_tr 0.787 | 0.464 (11위) | 0.323 | 정상 |

**판정 기준은 절대 점수가 아니라 "random 최하 + 갭"이다.** random 이 5벤치 전부
꼴찌이고 갭이 0.3+ 이면 벤치가 제 역할을 한다(멍청한 탐색과 똑똑한 탐색을 가름).
random 이 상위권으로 올라오면 그건 목적 충돌(F6)이 약하거나 good 영역이 넓다는
신호 — 과거 "포화" 사건의 지문이다.

> chebyshev score = 1 은 "완벽"이 아니라 "전 목적 동시 최상위"이며, F6 때문에
> 이 벤치들에선 사실상 도달 불가능하다. 절대 품질은 score 1 이 아니라
> **true optimum 참조선**(챔피언 무노이즈 100K + 2-swap 폴리시)과 비교한다.
