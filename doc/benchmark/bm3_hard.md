# bm3_hard — X → raw_y 구성

> 교차-블록 상호작용 + 기만 + ruggedness 의 **복합** 난이도. 여러 factor 가
> 동시에 걸려 가장 어렵다. `_structure_seed = 303`.
> set2(15컬럼)가 병목이라 group2 목적의 유효차원 25 가 승부처.

공통 파이프라인은 `stage.md` §0 참조. Easy/Medium 과 달리 단봉 골격 비중을
**0.7 배로 낮춰** 난이도 성분의 영향력을 키운다.

## latent 함수

```
latent_k(u) = 0.7·Σ_c w_kc·(1 − (u_c − p_kc)²)         ① 단봉 골격 (비중↓)
            + 0.05·Σ_{8쌍} s·u_a·u_b                    ② 블록 내 pairwise  (F2)
            + Σ_{10쌍} w×·cos(π(u_common − u_set))       ③ 교차-블록        (F3)
            + Σ_{3컬럼} w†·g(u)                           ④ 기만            (F4)
            + Σ_c r·sin(15~25·u_c + φ)                    ⑤ rugged          (F5)
            + c_gain_k · c(u)                             ⑥ trade-off       (F6)
```

### ③ 교차-블록 상호작용 (F3) — 이 벤치의 핵심

- 목적당 **10쌍**의 (common 컬럼, 자기 set 컬럼)에 `w×·cos(π(u_c − u_s))`,
  `w× ~ U(0.03, 0.07)`.
- `cos(π(u_c − u_s))` 는 common 값 u_c 가 바뀌면 set 쪽 최적 위치 u_s 가
  **함께 이동**하게 만든다. → "common 을 먼저 고정하고 set 을 최적화" 식
  블록 분해가 함정에 빠진다. common 을 매 라운드 재방문해야 한다.

### ④ 기만 (F4, 약)

```
g(u) = 0.35·u + 1.2·max(0, u − 0.85)/0.15
```
- u<0.85 구간에선 완만한 오르막(가짜 정상 u≈0.8 로 유도), 진짜 정상은 u≈1
  근처 **폭 0.15 의 급경사** 위. 낮은 해상도 탐색·1-hop 언덕오르기를 저격.
- 목적당 3컬럼, `w† ~ U(0.10, 0.16)`.
- **주의(정직 노트)**: g(u)는 단조증가라 backbone 과의 합에서만 기만이 생기는
  **약한 기만**이다(경계 선호 지형에 가까움). 강한 기만은 bm4 의 진짜 trap.

### ⑤ ruggedness (F5)

- `sin`, `freq ~ U(15,25)` (고주파), 진폭 ≈ 0.02 — **노이즈(5%)와 구분하기
  어려운 잔물결**. 미세 개선/악화 판정을 불안정하게 만든다.
- bm2 의 저주파 다봉(넓은 basin)과 대비: 이건 좁고 촘촘한 잔물결.

### ⑥ trade-off

`c_gain ~ U(0.35, 0.5)` — 강한 충돌.

## 왜 어려운가 (설계 의도)

1. F3 때문에 블록을 따로 최적화하면 common 이 움직일 때마다 set 최적이 무효화.
2. F4 때문에 국소 개선만 따라가면 가짜 정상(u≈0.8)에 수렴.
3. F5+노이즈 때문에 미세 판정이 불안정.
4. set2 15컬럼 → group2 유효차원 25 가 병목.
