# bm2_medium — X → raw_y 구성

> Easy 의 단봉 골격 위에 **블록 내 상호작용 + 완만한 다봉**을 얹은 벤치마크.
> 지역최적이 여러 개 생기지만 basin 이 넓어 population 계열(GA/PSO/ACO)이
> 헤쳐나갈 수 있는 수준. `_structure_seed = 202`.

공통 파이프라인은 `stage.md` §0 참조.

## latent 함수

```
latent_k(u) = Σ_c w_kc·(1 − (u_c − p_kc)²)          ① 단봉 골격 (bm1 과 동일)
            + 0.05·Σ_{8쌍} s·u_a·u_b                 ② 블록 내 pairwise  (F2)
            + Σ_c sin_w·sin(freq_c·u_c + φ_c)        ③ 저주파 다봉      (F5')
            + c_gain_k · c(u)                         ④ trade-off        (F6)
```

### ② 블록 내 pairwise 상호작용 (F2)

- 목적마다 자기 그룹 컬럼에서 **8쌍**(a,b)을 뽑고 `s·u_a·u_b`(부호 s=±1 랜덤)을
  더한다. 진폭 0.05/쌍 → 목적당 총 ±0.4 수준.
- 효과: 컬럼 a 의 최적값이 컬럼 b 의 현재값에 **의존**하게 된다 → 완전한
  좌표 독립이 깨진다. 단 진폭이 작아 단봉 골격이 여전히 지배적.

### ③ 저주파 다봉 (F5')

- `sin(freq·u + phase)`, `freq ~ U(2,4)·π` (저주파), 진폭 0.15.
- 효과: 매끄러운 단봉 위에 **넓고 완만한 물결**을 얹어 지역최적을 여러 개
  만든다. 저주파라 basin 이 넓어 — 개체군이 여러 basin 을 동시에 탐색하면
  탈출 가능("medium"). (bm3 의 고주파 rugged F5 와 대비: 이건 넓은 다봉.)

### ④ trade-off (F6)

`c_gain ~ U(0.3, 0.45)` — bm1(0.25–0.35)보다 약간 강한 충돌.

## 왜 중간 난이도인가

- 상호작용(F2)이 좌표 독립을 깨지만 약하고, 다봉(F5')이 지역최적을 만들지만
  basin 이 넓다. → greedy 는 지역최적에 갇히기 쉽지만, 개체군/모델 기반은
  넓은 basin 을 활용해 넘어간다.
- 저격 대상: "좌표를 하나씩 최적화" 가정. 활용 유리: GA/PSO/ACO 의 다점 탐색.

## raw_y 로의 변환

stage.md §0-④ 동일 — latent 6값 → mask1(y11,y12) + mask2(y21,y22) + 스칼라(y13,y23).

## 실측 (budget 780, 10 seed)

챔피언 xgb_tr 0.833 / random 0.485(11위) / 갭 0.348. 다섯 벤치 중 챔피언
점수가 가장 높다 — 다봉·상호작용이 있어도 basin 이 넓어 잘 찾으면 높은 해에
도달하기 때문.
