# bm4_deceptive — X → raw_y 구성

> **기만 지배형** — 진짜 trap 함수. bm3 의 약한 g(u) 와 달리 국소 정보가
> 전부 틀린 방향(가짜 정상)을 가리키고, 진짜 최적은 골짜기 너머 좁은 곳에
> 있다. 상호작용은 최소화 → "탐색 능력"이 아니라 **탈출 능력**이 승부를 가른다.
> `_structure_seed = 404`.

공통 파이프라인은 `stage.md` §0 참조.

## latent 함수

```
latent_k(u) = 0.7·Σ_c w_kc·(1 − (u_c − p_kc)²)      ① 단봉 골격 (비중↓)
            + 0.05·Σ_{4쌍} s·u_a·u_b                 ② 블록 내 pairwise (약, F2)
            + Σ_{6컬럼} w†·trap(u)                    ③ 진짜 trap        (F4)
            + c_gain_k · c(u)                         ④ trade-off        (F6)
```

교차-블록(F3)도 rugged(F5)도 **없다.** 지배 성분은 ③ trap 하나.

### ③ 진짜 trap (F4) — 이 벤치의 핵심

```
trap(u) = (v − u)/v            (u ≤ v)   가짜 정상 u=0 을 향한 오르막, 높이 1.0
        = 1.4·(u − v)/(1 − v)  (u > v)   골짜기 너머 진짜 정상 u=1, 높이 1.4
v = 0.85 (골짜기 위치)
```

- 고전적 쌍봉 trap: u 를 **낮추는** 방향(u=0)이 국소적으로 좋아 보이지만,
  진짜 최적은 골짜기(u≈0.85)를 **점프**해 건너야 하는 u=1 에 있다(높이 1.4).
- 1-hop 이웃 이동·greedy·유한차분 gradient 는 전부 u=0 으로 끌려간다.
  골짜기를 뛰어넘는 **랜덤 점프 mutation / restart / 전역 탐험**만이 탈출한다.
- 목적당 **6컬럼**, `w† ~ U(0.10, 0.16)`.
- **cardinality ≥ 4 컬럼만** trap 대상 — card 2 는 양 끝(u=0,u=1)을 다 보므로
  trap 이 성립하지 않는다(점프가 곧 이웃이라 골짜기가 없음).

### ② 블록 내 pairwise (약)

bm3 의 절반인 4쌍. trap 은 컬럼별 독립(가법적)이라 **상호작용 모델링 능력은
도움이 안 된다** — 이 벤치가 순수하게 "탈출 능력"만 측정하도록 상호작용을 줄임.

## 왜 어려운가

- 국소 정보가 100% 틀린 방향을 가리킴 → gradient/greedy 계열 무력.
- trap 이 가법적이라 tree surrogate 의 상호작용 학습도 큰 이점 없음.
- 승부처: 골짜기를 건너뛰는 탐험(SA 의 점프 수락, GA/ACO 의 랜덤 재조합,
  xgb_tr 의 restart+trust-region 재시드).
