# bm5_epistasis — X → raw_y 구성

> **epistasis(상호작용) 지배형** — 강한 교차-블록 결합으로 심하게 비분리
> (non-separable). 지형은 매끄럽지만(기만·rugged 없음) 어떤 컬럼의 최적값도
> 다른 컬럼들에 조건부다. 좌표/marginal 계열의 급소. `_structure_seed = 505`.

공통 파이프라인은 `stage.md` §0 참조. 단봉 골격 비중을 **0.5 배**까지 낮춰
상호작용이 지형을 지배하게 한다(5벤치 중 가장 낮음).

## latent 함수

```
latent_k(u) = 0.5·Σ_c w_kc·(1 − (u_c − p_kc)²)       ① 단봉 골격 (비중 최소)
            + 0.08·Σ_{12쌍} s·u_a·u_b                 ② 블록 내 pairwise (강, F2)
            + Σ_{20쌍} w×·cos(π(u_common − u_set))     ③ 교차-블록 (강, F3)
            + c_gain_k · c(u)                          ④ trade-off        (F6)
```

기만(F4)도 rugged(F5)도 **없다** — 순수하게 상호작용만으로 어렵게 만든다.

### ② 블록 내 pairwise (F2, 강)

- **12쌍**(bm3 의 1.5배), 진폭 **0.08**(bm3 의 0.05보다 강). 같은 블록 안
  변수쌍 결합을 크게 키운다.

### ③ 교차-블록 상호작용 (F3, 강) — 이 벤치의 핵심

- 목적당 **20쌍**(bm3 의 2배), `w× ~ U(0.06, 0.12)`(bm3 의 0.03–0.07 의 약 2배).
- `cos(π(u_common − u_set))` — common 값이 바뀌면 set 쪽 최적이 크게 이동.
  쌍 수·가중치가 모두 2배라 **비분리성이 지배적**이다.
- 효과: 한 컬럼만 바꿔 개선하려는 좌표 하강은 다른 컬럼들의 최적을 계속
  무효화시켜 진전하지 못한다. 블록 단위로 **함께** 움직여야 개선된다.

### ④ trade-off

`c_gain ~ U(0.35, 0.5)` — bm3/bm4 와 동일.

## 왜 어려운가 / 무엇을 저격하나

- 매끄럽지만 심하게 비분리 → 좌표/marginal(coordinate, ACO, TPE 의 컬럼독립
  가정)이 무너진다.
- 유리한 전략: 상호작용 학습(xgb tree surrogate) 또는 블록 단위 재조합
  (GA 의 블록 crossover, GOMEA 의 FOS 이식).
- **주의 — 탐색 난이도 ≠ 목적 충돌**: bm5 는 "찾기 어렵게"(비분리) 만든 벤치지
  "전 목적 동시만족 불가"(강한 F6)를 키운 벤치가 아니다. 그래서 잘 찾는
  optimizer 는 높은 점수까지 도달할 수 있다(상한이 다른 벤치보다 높게 형성).
  실측에서 챔피언 0.787 로 정상 범위지만, 만약 어떤 구현이 0.96 처럼 크게
  높다면 그건 "버그"가 아니라 그 구현의 **목적 충돌이 약한 것**이다 — 판정은
  절대 점수가 아니라 random 최하 여부로 한다(stage.md §3).

## 실측 (budget 780, 10 seed)

챔피언 xgb_tr 0.787 / random 0.464(11위) / 갭 0.323. 상위권이 ga·xgb_surrogate
(0.767)로 근접 — 상호작용을 다루는 계열(tree 학습·블록 재조합)이 강함을
보여준다. 좌표독립 계열(eda_tree 0.535, aco 0.585)이 하위로 밀리는 것도
epistasis 지배의 예상된 지문.
